    # 1. SDK嵌入

    1. SDK下载完，将aar的 common.aar login.aar anti.aar pay.aar alipaySdk.aar 类库导入游戏工程libs目录下

    注意点：有接入1.1.0.0之前版本请先删除hykb_sdk.aar和alipaySdk.aar

    1. 在主工程的build.gradle文件下添加以下代码即可完成依赖，
    android {
    defaultConfig {
       //方法数超过65536时，开启多dex的支持
       multiDexEnabled true
    }
    
    repositories {
      flatDir {
        dirs 'libs'
    }
  }
}

dependencies {
    //引入SDK依赖
    implementation(name:'common', ext:'aar')
    implementation(name:'login', ext:'aar')
    implementation(name:'anti', ext:'aar')
    implementation(name:'pay', ext:'aar')
    implementation(name:'alipaySdk', ext:'aar')
}

    # 2. 接入代码

    # 2.1 SDK初始化

    描述：

    初始化接口必须在所有SDK的其他相关api调用之前进行调用。

    接口方法:

    //SDK初始化接口
HykbLogin.initSdk(Activity activity, String gameId, int orientation,  HykbInitListener listener)
    1
    2

    参数定义：

    参数名称 描述
    activity 当前游戏的主Activity
    gameId 快爆开发者前台创建游戏后，获得的游戏ID
    orientation 游戏屏幕方向，整型，0:横屏，1:竖屏
    listener HykbInitListener 初始化回调监听

    示例代码：

    /**
* SDK初始化方法
*/
HykbLogin.initSdk(MainActivity.this, GAME_ID, ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE,new HykbInitListener() {

      @Override
      public void onInitSuccess(final HykbUser user) {
        if (user == null) {
          ToastUtil.showToast(MainActivity.this, "初始化成功 user = null ");
        } else {
          ToastUtil.showToast(MainActivity.this, "初始化成功 user = " + user.toString());
        }
      }

      @Override
      public void onInitError(final int code, final String msg) {
        HandlerUtils.runOnMainThread(new Runnable() {
          @Override
          public void run() {
            if (1102 == code) {
              //1102：表示维护中，的退出游戏按钮回调
              MainActivity.this.finish();
            } else if (2005 == code) {
              //2005：表示需要游戏做退出操作
              MainActivity.this.finish();
            } else {
              ToastUtil.showToast(MainActivity.this, msg);
            }
          }
        });

      }

      @Override
      public void onSwitchUser(boolean success, int code, HykbUser user) {
	    //这边的切换账号是来自，默认登录的切换账号的回调
        if (success) {
          ToastUtil.showToast(MainActivity.this, "切换成功 user = " + user.toString());
        } else {
          ToastUtil.showToast(MainActivity.this, "账号未切换 ");
        }
      }
    });

    类描述：

    初始化监听类： com.m3839.sdk.login.listener.HykbInitListener

    方法名称 描述
    onInitSuccess(HykbUser user) 初始化成功后回调方法，user不为空，表示账号在有效期内，自动登录
    onInitError(int code, String msg) 初始化失败回调方法，包含了防沉迷提示退出游戏回调操作
    onSwitchUser(boolean success, int code, HykbUser user) 在自动登录的情况下，进行账号切换的回调

    检查sdk是否初始化成功

    boolean isInitOk = HykbLogin.isInitOk()

    # 2.3 登录

    接口方法:

    /**
 * 登录api的调用
 */
HykbLogin.login(Activity activity, HykbLoginListener listener);
    1

    参数定义：

    参数名称 描述
    activity 当前游戏主页面的Activity
    listener HykbLoginListener 登录回调监听

    示例代码:

        if (HykbLogin.getUser() != null) {
      ToastUtil.showToast(MainActivity.this, "登录成功 user = " + HykbLogin.getUser().toString());
      return;
    }
    HykbLogin.login(MainActivity.this, new HykbLoginListener() {
      @Override
      public void onLoginFinished(boolean success, int code, HykbUser user) {
        if (success) {
          ToastUtil.showToast(MainActivity.this, "登录成功 user = " + user.toString());
        } else if(code == 2005){
           MainActivity.this.finish();
        } else {
          ToastUtil.showToast(MainActivity.this, "登录失败 code = " + code);
        }
      }
    });

    类描述：

    登录监听类： com.m3839.sdk.login.listener.HykbLoginListener

    方法名称 描述
    onLoginFinished(boolean success, int code, HykbUser user) 登录回调监听

    注意点: 在登录code=2003，考虑到内购SDK无“去认证”弹窗进行承载取消授权登录后续的逻辑，且以强杀游戏进程的方式进行处理，用户容易误以为是游戏闪退，取消授权登录的逻辑，仍以回调的方式给游戏方进行自主处理

    内购SDK接入文档需新增这部分回调的说明 游戏方需针对该回调弹出重新登录弹窗

    参考弹窗文案：很抱歉，您还未登录，请您登录之后进行游戏 参考弹窗按钮：立即登录 点击按钮重新唤起快爆登录授权页面

    # 2.7 支付

    # 2.9 回调code的说明：

    code 描述
    -1 服务端请求异常
    -2 未进行初始化，调用了支付或者登录
    -3 未进行登录，调用了支付
    -4 支付参数商品名称空了
    -5 支付参数商品名称超过16个字符
    -6 支付参数商品金额必须要大于0
    -7 支付参数商品金额最大不超过五万
    -8 cpOrderId不能为空
    -9 cpOrderId不能超过128位
    1101 游戏不存在，未进行审核，联系运营人员处理
    1102 游戏开发者需要，退出游戏操作或者注销账号
    2001 用户认证失败
    2002 登录失败
    2003 取消登录
    2004 登录异常
    2005 游戏开发者需要，退出游戏操作或者注销账号
    2006 重新调用登录
    2101 支付未配置，联系运营人员处理
    2102 充值维护中
    2103 CP订单号错误
    2104 订单不存在
    2105 充值失败
    2160 支付成功，白名单的测试支付返回
    2161 支付失败，白名单的测试支付返回
    2199 未查询到订单状态
    3001 支付弹窗关闭
    3002 支付限额弹窗关闭
    3004 支付重试弹窗关闭
    4000 系统异常
    4001 数据格式不正确
    6001 支付取消
    8000 支付结果确认中
    9000 支付成功

    # 2.10 代码混淆

    如果您需要使用proguard混淆代码，需确保不要混淆SDK的代码。 请在proguard.cfg文件(或其它混淆文件)尾部添加如下配置:

    # 不能混淆监听
-keep class com.m3839.sdk.common.**{*;}
-keep class com.m3839.sdk.anti.**{*;}
-keep class com.m3839.sdk.user.**{*;}
-keep class com.m3839.sdk.login.**{*;}
-keep class com.m3839.sdk.pay.**{*;}
-keep class com.alipay.sdk.app.PayTask{ public *;}
-keep class org.json.alipay.** { *; }
-keepclasseswithmembernames class com.m3839.sdk.common.js.JsInterface {
    <methods>;
}

# Prevent R8 from leaving Data object members always null
-keepclassmembers,allowobfuscation class * {
  @com.google.gson.annotations.SerializedName <fields>;
}
