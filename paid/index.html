# 1. SDK嵌入

# 1.1 Android Studio集成方式

  1. SDK下载完，将aar的 common.aar anti.aar paid.aar 类库导入游戏工程libs目录下

注意点：有接入1.1.0.0之前版本请先删除m3839_paid_checker.aar

  1. 在主工程的build.gradle文件下添加以下代码即可完成依赖，
android {
    defaultConfig {
       //方法数超过65536时，开启多dex的支持
       multiDexEnabled true
    }
    
    repositories {
      flatDir {
        dirs 'libs'
    }
  }
}

dependencies {
    //引入SDK依赖
    implementation(name:'common', ext:'aar')
    implementation(name:'anti', ext:'aar')
    implementation(name:'paid', ext:'aar')
}

# 2. 接入代码

# 2.1 付费校验

描述：

SDK的检测方法。

接口方法:

HykbPaidChecker.checkLicense(activity, appId, publicKey, listener);
1

参数定义：

参数名称 描述
context 当前应用的Activity
appId 应用ID
publicKey 公钥
listener HykbCheckListener 检测回调监听

示例代码：

String publicKey = "申请的公钥";
String appId = "申请的appid";
HykbPaidChecker.checkLicense(activity, appId, publicKey,
        new HykbCheckListener() {
        @Override
        public void onAllowEnter() {
          //通过验证，进入游戏
        }

        @Override
        public void onReject(int code, String errorMsg) {
          finish();
        }
     });

类描述：

检测监听类： com.m3839.sdk.paid.HykbCheckListener

方法名称 描述
onAllowEnter() 检测成功后回调方法
onReject(int code, String errorMsg) 检测失败回调方法

常见回调的code描述：

code 描述
20001 用户手动关闭
20002 没有安装快爆
20003 没有购买
20004 服务端数据异常
401 参数不完整
403 token异常
-1 防沉迷退出游戏

# 2.2 代码混淆

如果您需要使用proguard混淆代码，需确保不要混淆SDK的代码。 请在proguard.cfg文件(或其它混淆文件)尾部添加如下配置:

-keep class com.m3839.sdk.common.**{*;}
-keep class com.m3839.sdk.anti.**{*;}
-keep class com.m3839.sdk.user.**{*;}
-keep class com.m3839.sdk.paid.**{*;}
-keepclasseswithmembernames class com.m3839.sdk.common.js.JsInterface {
    <methods>;
}
