# 1. Import SDK

# 1.1 Integration

  1. DownloadSDKfirst, then copy common.aar archives.aar into your main module/libs.

  2. Edit your main module build.gradle file，add the following code:

repositories {
  flatDir {
      dirs 'libs'
  }
}
dependencies {
    implementation(name:'common', ext:'aar')
    implementation(name:'archives', ext:'aar')
}

# 2. Development

# 2.1 introduction to gamearchives databean data entity class

Field meaning Field Name Optional Description
Game ID gameId must The cloud archive is the only character in the game access file.
Use this ID to reference games saved in your game client.
User ID userId must Hykbuser obtained after successful login from HYKB anti addiction SDK or payment SDK
Archive ID archivesId must The gear id provided by the game developer must be an integer
Archive Title archivesTitle must The archive Title provided by the game developer is a string with a length of no more than 30
Archive Content archivesContent must The content size of the archive Title provided by the game developer shall not exceed 3M
Save Archive Time updateTime auto After successful archiving, this field is automatically generated by the server

For the above field contents, set the value through set，such as setGameId("111"), get method value，such as getArchivesContent()

Important attention： All API interfaces of the SDK have a request interval limit of 3 seconds The content size of archive content shall not exceed 3M

# 2.2 Save Archive

Sample：

GameArchivesDataBean dataBean = new GameArchivesDataBean();
dataBean.setGameId("Game ID");
dataBean.setUserId("Hykb user ID");
dataBean.setArchivesId(11);//archive ID，integer，by the developer define
dataBean.setArchivesTitle("Archive title");
dataBean.setArchivesContent("Archive Content");
HykbGameArchives.saveArchivesData(SaveArchivesActivity.this, dataBean, new HykbSaveArchivesListener() {
                @Override
                public void onSuccess() {
                    //Save Archive success
                }

                @Override
                public void onFailed(int code, String message) {
	                //Save Archive failed，reason is in code and message
                    Log.i("archives","code = "+code+",message = "+message);
                }
            });

# 2.3 Read Archive

Sample：

GameArchivesDataBean dataBean = new GameArchivesDataBean();
dataBean.setGameId("Game ID");
dataBean.setUserId("Hykb User ID");
dataBean.setArchivesId(11);//archive ID，integer，by the developer define
HykbGameArchives.readArchivesData(SaveArchivesActivity.this, dataBean, new HykbReadArchivesListener() {
    @Override
    public void onSuccess(GameArchivesDataBean dataBean) {
        //Read Archive success
        StringBuilder sb = new StringBuilder();
        sb.append("Archive ID：");
        sb.append(dataBean.getArchivesId());
        sb.append("\n");
        sb.append("Archive Title：");
        sb.append(dataBean.getArchivesTitle());
        sb.append("\n");
        sb.append("Archive Content：");
        sb.append(dataBean.getArchivesContent());
        Log.i("archive", sb.toString());
    }

    @Override
    public void onFailed(int code, String message) {
        // Read Archive failed，reason is in code and message
        Log.i("archive","code = "+code+", message = "+message);
    }
});

# 2.5 Proguard

If you are using Proguard, add the following rules to your ProGuard configuration file:

-keep class com.m3839.sdk.common.** {*;}
-keep class com.m3839.sdk.archives.** {*;}

# 2.6 Code and Message Description

code message
101 Archive Title exceeds 30 characters
102 Network error
103 Operation is too frequent (at present, the interval between archiving, file reading and list request is 3S)
1001 Archive save failed
1101 The account information has expired. Please log in again
2001 Game archiving is not configured (currently, it can only be used after the game ID is configured by technology)
2002 Archive does not exist
2004 Archive content exceeds 1m
2005 A maximum of 10 gears can be stored
2101 Package body error (check whether the package name and signature are correct)
